# Charts & plots — catalog

Informative graphs for the MCAT-Prep evidence. All charts are **hand-rolled SVG,
pure Python stdlib** (no matplotlib/numpy), so they regenerate offline and version
cleanly. Two generators:

- The eval scripts embed **their own** chart when they run
  (`calibration.py` -> `memory-calibration.svg`, `ablation.py` -> `ablation.svg`,
  `bench.py` -> `bench-latency.svg`).
- `evals/plots.py` renders the **rest** from the committed datasets / fixtures /
  reported numbers (so they stay in sync with the canonical files).

Regenerate everything:

```sh
python3 evals/plots.py            # the 6 charts below marked "plots.py"
python3 evals/calibration.py      # memory-calibration.svg
just ablation                     # ablation.svg (+ ABLATION.md)
just bench                        # bench-latency.svg (+ BENCHMARK.md)
```

## Possible future charts (data-limited or not yet built)

These would be informative but need data or work we have not done yet — listed so
the gap is explicit and honest:

- **Memory -> performance gap** (the core thesis): per-topic card recall vs
  reworded-question accuracy. Needs the **paraphrase test** (rubric §7d, 30 cards ×
  2 reworded), which requires human/tester answers — not yet run.
- **Knowledge-graph / concept lattice** (the 172-KC prerequisite DAG) as a static
  layered diagram, mirroring the live desktop/Android lattice. Doable from the KC
  map; deferred (the apps render it live).
- **Coverage vs the AAMC outline** per section (bar of % covered) — needs the §8
  outline mapping, which is not yet surfaced on the dashboard.
- **Benchmark latency distribution** (p50/p95/worst per action, log-scale) — a
  richer view than the current p95-vs-target bar; easy follow-up in `plots.py`.
- **Per-seed ablation dot plot** (12 seeds per arm) — shows the ON<OFF separation
  holds on every seed; the fixture already has per-seed `values`.
- **Sync-conflict / crash-recovery result charts** — once `just crash-test` /
  `just offline-test` and a sync-conflict test produce numbers (rubric §6/§7).

## Style notes (for adding a chart)

- Match `evals/calibration.py::build_svg`: ~640x500 viewBox, `Helvetica,Arial`,
  white background; palette title `#111827`, labels `#6b7280`, gridlines `#eef2f7`,
  border `#9ca3af`, axes `#374151`, data `#2563eb`; qualitative series add
  `#16a34a` / `#d97706` / `#dc2626` / `#9ca3af`.
- ASCII/entities only inside the SVG (`--`, `&lt;=`); validate with
  `xml.dom.minidom.parseString` before writing.
- Label synthetic data as SYNTHETIC and keep numbers honest / in sync with the
  canonical `evals/*.md`.
